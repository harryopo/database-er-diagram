# Database ER Diagram Skill

> 专业的数据库实体关系图（ER Diagram）生成工具，严格遵循 Oracle Academy 标准

## 概述

本 Skill 通过draw\.io生成高质量的数据库概念模型 ER 图，基本遵循深圳信息职业技术大学计算机应用技术数据库设计与实现绘制er图的教程，集中提取老师的er图绘制教材的规范。并加之以拓展，旨在方便更清晰显示和便于阅读。er图支持 Crow's Foot（鸦脚）记法、NT 不可转移标记、交集实体、递归关系等所有标准元素。

## 特性

| 特性                       | 说明                        |
| ------------------------ | ------------------------- |
| 📐 **Oracle Academy 标准** | 严格遵循 DFo 2-6/3-1/3-2 教材规范 |
| 🦶 **Crow's Foot 记法**    | 专业的 ERone/ERmany 基数标注     |
| **颜色系统**                 | 绿色=必须(MUST) / 粉色=可选(MAY)  |
| 🔷 **NT 菱形标记**           | 不可转移关系的 3 段 edge 实现       |
| 📦 **交集实体**              | M:N 关系自动解析为交集实体           |
| 🔄 **递归关系**              | 同一实体内的自引用关系               |
| 📊 **多种布局**              | 三列紧凑布局 / 星型分布布局           |
| **属性标记**                 | `#`主键 / `*`必填 / `o`可选     |

## 快速开始

### 安装/环境配置

1. 将此skill安装至全局或项目中
2. 为方便查看和编辑改动，最好安装 VS Code 或IDE环境中的插件：`hediet.vscode-drawio`
3. 当然，你可以在在线网站 draw\.io 中打开文件进行编辑查看

### 使用

在绘制er图之前，需在对话中描述你的数据库设计需求，例如：

```
请为我的在线书店设计ER图，包含以下实体：
- 用户（USER）：id、用户名、邮箱
- 书籍（BOOK）：id、标题、作者、价格
- 订单（ORDER）：id、用户ID、下单时间、总价

关系：
- 一个用户可以下多个订单
- 一个订单包含多本书
- 每本书有且仅有一个作者
```

Skill 将生成完整的 `.drawio` 文件，如果安装/环境配置正确，你可以：

- 在 VS Code 或其它IDE环境中直接预览
- 导出为 PNG/SVG/PDF
- 用 draw\.io 在线编辑

## 核心规范速查

### 实体类型配色

| 类型   | 颜色    | fillColor | 示例               |
| ---- | ----- | --------- | ---------------- |
| 核心实体 | 蓝色    | `#dae8fc` | USER, ORDER      |
| 次要实体 | 🟢 绿色 | `#d5e8d4` | USER\_ADDRESS    |
| 角色实体 | 🟡 黄色 | `#fff2cc` | CARE\_PROVIDER   |
| 业务实体 | 🟣 紫色 | `#e1d5e7` | PET              |
| 结果实体 | 🔴 红色 | `#f8cecc` | REVIEW           |
| 日志实体 | 灰色    | `#f5f5f5` | EXCEPTION\_EVENT |
| 交集实体 | 🟠 橙色 | `#ffe6cc` | TASK\_PET        |

### 关系连线

| 类型            | 线型         | 颜色           | 含义   |
| ------------- | ---------- | ------------ | ---- |
| 必须参与（MUST BE） | `━━━` 实线   | `#7BC8A4` 绿色 | 至少1个 |
| 可选参与（MAY BE）  | `- - -` 虚线 | `#F4A8B0` 粉色 | 可以0个 |

### 基数标注

| 符号  | 名称     | 含义                 |
| --- | ------ | ------------------ |
| `│` | ERone  | Exactly ONE（恰好一个）  |
| `<` | ERmany | ONE OR MORE（一个或多个） |

### 属性标记

| 符号   | 含义      | 示例           |
| ---- | ------- | ------------ |
| `#`  | 主键（UID） | `# user_id`  |
| `*`  | 必填      | `* username` |
| `o`  | 可选      | `o avatar`   |
| `#*` | 主键且必填   | `#* id`      |

### 读法规则（Oracle Academy 标准）

从**实体A**看**实体B**：

1. 看**靠近A端的线型** → 实线=MUST / 虚线=MAY
2. 看**靠近B端的基数符号** → 单线=ONE / 鸦脚=MANY

**示例**：`BOOK ━━━(实线)──(鸦脚在BOOK端) - - - AUTHOR`

- 从BOOK看AUTHOR：实线(MUST) + AUTHOR端单线(ONE) → "每本BOOK必须由一位AUTHOR编写"
- 从AUTHOR看BOOK：虚线(MAY) + BOOK端鸦脚(MANY) → "每位AUTHOR可能是一位或多位BOOK的作者"
